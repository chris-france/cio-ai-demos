Cost Analysis — Local vs Cloud

What it costs to run 1 million tokens through each model

Local Models (Free)

  • $0 per million tokens — you already own the hardware
  • No API keys, no rate limits, no usage tracking
  • Data never leaves your machine or network
  • One-time download, runs offline forever

Cloud APIs (Subscription or Per-Token)

  • $20/month subscription (ChatGPT Plus, Claude Pro)
  • API pricing: $3–$15 per million input tokens
  • $15–$75 per million output tokens
  • Best quality available, but data leaves your control

PROMPT

"Is this email spam or not spam: 'Congratulations! You've won a free iPhone...'"

BEST FIT: 1B LOCAL

Result: "Spam" — Correct. Simple binary classification works perfectly on the smallest model. Zero cost, zero latency, zero data exposure.

WHY NOT CLOUD?

Cloud gives the same answer but costs money and sends your email content to a third party. For classification, local is the smart choice.

PROMPT

"Summarize this 10-page contract into 5 key points for the executive team."

BEST FIT: 3B–8B LOCAL

The 3B model produces clean, structured summaries. The 8B model adds better nuance. Both run locally — your contract text never leaves the building.

WHY NOT 1B?

The 1B model misses context and produces vague bullets. For multi-page documents, you need at least 3B parameters to capture the key details.

PROMPT

"Analyze our Q3 financials, identify anomalies, compare to industry benchmarks, and recommend 3 strategic adjustments with risk assessments."

BEST FIT: CLOUD (CLAUDE / GPT-4)

Multi-step reasoning with industry knowledge requires the largest models. The cloud model spots trends the local models miss entirely.

WHEN TO USE CLOUD

High-stakes decisions, multi-step reasoning, and tasks requiring broad knowledge. Worth the $20/month when the output informs a $500K+ decision.

PROMPT

"Write a Python script that connects to our PostgreSQL database, pulls the last 30 days of transactions, and generates a PDF report."

BEST FIT: CLOUD OR 8B LOCAL

The 8B model writes functional code. Cloud models write production-quality code with error handling, logging, and edge cases covered.

THE TRADEOFF

For internal scripts — use 8B local (free, private). For customer-facing code or complex integrations — use cloud. The $0 vs $20 question depends on who sees the output.
