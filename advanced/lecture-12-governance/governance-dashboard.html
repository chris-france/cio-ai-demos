Meridian Health Systems — AI Acceptable Use Policy

DRAFT v1.0 — Auto-generated March 15, 2026 | Effective: [Pending Legal Review]

▶ 1. Purpose and Scope

This policy governs the use of all artificial intelligence tools, platforms, and services by Meridian Health Systems employees, contractors, and affiliates. It applies to 10 currently inventoried AI tools and any future AI tools adopted by the organization.

The policy aims to maximize the productivity benefits of AI while protecting patient data, intellectual property, and organizational reputation.

▶ 2. Approved AI Tools

Only the following AI tools are approved for use on company data:

  • • ChatGPT Enterprise (Engineering) — High-sensitivity data approved with DLP controls
  • • GitHub Copilot (Engineering) — Source code only; API keys must never be in scope
  • • Grammarly Business (Marketing) — Internal communications; no PHI
  • • Jasper AI (Marketing) — Brand content only; no client data
  • • Claude Pro (Executive Team) — Strategy docs; no PHI or financial actuals
  • • Notion AI (Product) — Internal wikis; no restricted data
  • • HuggingFace Inference (Data Science) — Approved models only; no PII in training data

Pilot: Cursor (Engineering, 5 users) — under evaluation through Sprint 3

Using any AI tool not on this list is a violation of this policy.

▶ 3. Data Classification Requirements

▶ 4. Prohibited Activities
  • ✗ Uploading patient health information (PHI) to any AI tool
  • ✗ Using personal AI subscriptions for work purposes
  • ✗ Sharing API keys, passwords, or credentials in AI prompts
  • ✗ Using AI-generated content in client deliverables without human review
  • ✗ Disabling DLP or security controls on approved tools
  • ✗ Using AI to make autonomous decisions about patient care
  • ✗ Training external models on proprietary company data
▶ 5. IP Ownership and AI-Generated Content

All AI-generated content created using company tools and data is the property of Meridian Health Systems. Employees must:

  • • Document AI involvement in any deliverable
  • • Verify AI-generated code for licensing compliance
  • • Not claim AI-generated work as solely human-authored
  • • Report any suspected IP infringement in AI outputs
▶ 6. Enforcement and Violations

Violations of this policy will be handled through the standard disciplinary process. Shadow AI usage will be treated as an unauthorized data access incident. Repeat violations may result in termination of AI tool access and/or employment action.

Report suspected violations to: ai-governance@meridianhealth.com
